# NebulaOps v22.5 — APIForge Kubernetes Integration

## Scope

NebulaOps v22.5 adds APIForge as an App Bar entry and deploys it as a Kubernetes workload instead of embedding it as a shell micro frontend.

## Runtime exposure

- App Bar entry: `APIForge`
- Browser URL: `http://localhost:31110/apiforge/`
- Kubernetes namespace: `nebulaops`
- Kubernetes service: `apiforge`
- NodePort: `31110`
- Container port: `8080`
- Spring context path: `/apiforge`
- Health endpoint: `/apiforge/actuator/health`

## Files added

- `extensions/apiforge/` — Spring Boot APIForge application copied into the NebulaOps workspace.
- `infrastructure/kubernetes/apiforge.yaml` — namespace, PVC, deployment, service and ingress path.
- `scripts/wsl/deploy-apiforge-k8s.sh` — builds the image, loads it into kind when applicable and applies the manifest.
- `scripts/wsl/undeploy-apiforge-k8s.sh` — removes the runtime resources while preserving the PVC by default.

## Startup behavior

`./scripts/wsl/start.sh` now deploys APIForge to Kubernetes by default. Use `--skip-apiforge-k8s` to start only the Docker Compose platform and deploy APIForge later.

```bash
./scripts/wsl/start.sh --rebuild
./scripts/wsl/start.sh --rebuild --skip-apiforge-k8s
./scripts/wsl/deploy-apiforge-k8s.sh
```

## Kubernetes lifecycle

```bash
kubectl -n nebulaops get pods,svc apiforge
kubectl -n nebulaops logs deploy/apiforge
curl http://localhost:31110/apiforge/actuator/health
```

To remove APIForge but keep collections/history:

```bash
./scripts/wsl/undeploy-apiforge-k8s.sh
```

To remove APIForge and delete persisted data:

```bash
./scripts/wsl/undeploy-apiforge-k8s.sh --delete-data
```
