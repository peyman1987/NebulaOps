# NebulaOps v23.2 — Live-Only and Compile Hardening Report

## Scope

This update hardens the v23.2 package after the runtime-stability upgrade. The focus is to remove remaining bundled mock/static operational assumptions, keep UI/user-facing text in English, and add stronger preflight guards before runtime start.

## Implemented changes

### Live-only runtime guard

- Added `scripts/verify-live-only-runtime.py`.
- Added the guard to `scripts/wsl/preflight-v23.2.sh`.
- The guard rejects bundled demo endpoints, legacy seeded runtime records, personal placeholder accounts, offline snapshot JSON files and unsafe command-runner patterns.
- APIForge package data files are verified to start empty: collections, environments and history.

### Backend command runner hardening

- Updated all backend `ProcessExecutor` implementations to drain stdout/stderr asynchronously while enforcing timeout.
- Updated gateway `ToolCommandClient` with the same timeout-safe process handling.
- Updated extension runbook execution in KubeBridge and Contract Hub to drain process output asynchronously and avoid blocking on long-running scripts.

### Authentication cleanup

- Removed hard-coded local admin shortcut logic from `AuthController`.
- Local admin login is still available only when explicitly configured through external properties/environment variables.
- Removed the personal `peyman` user from the imported Keycloak realm.

### English-only UI cleanup

- Converted Spring MVC add-on templates, validation messages and controller flash messages to English.
- Converted Spring MVC deploy script messages/help text to English.
- Converted APIForge README/test report and Angular MFE README to English.
- Updated Keycloak branding from stale `legacy branding` to `N23.2`.

### Release hygiene

- Removed obsolete v22 shell compatibility assets.
- Renamed the stale v22 reverse-proxy diagram to the v23.2 naming scheme.
- Strengthened release validation for stale v22/previous release identifiers.
- Replaced outdated `default-org` examples in OpenAPI/API docs with `nebulaops`.

## Compile status

- Frontend Angular production build completed successfully.
- MFE remote compatibility checks completed successfully.
- Extension source-level Java guard completed successfully.
- Java syntax guards for updated process runners, gateway command client, auth controller and Spring MVC web controller/form completed successfully.

A full Maven reactor build was not executed in the packaging container because the `mvn` command and Maven wrapper are not available in this environment. The package includes the required WSL/Docker startup path so the full backend build can run in the target development environment through `./scripts/wsl/start.sh --rebuild`.
