# NebulaOps v23.1 — APIForge-only UI extension cleanup

## Scope

This package keeps APIForge as the only installed extension under `extensions/` and removes all non-installed extension entries from the APP BAR.

## Installed extension

```text
extensions/apiforge
```

## APP BAR behavior

The APP BAR now exposes one Extensions card only:

```text
APIForge -> /api/extensions/console
```

The console provides UI actions to start, stop, refresh status and open APIForge through the gateway.

## Removed UI entries

The following extension cards are not present in the frontend source or bundled dist artifact:

```text
KubeBridge
Runbook Center
Extension Registry
Contract Hub
EventOps Center
Observability Lens
GitOps Center
Secrets & Config Center
SLO Center
Backup & Recovery Center
```

## Verification

A new guard script was added:

```bash
./scripts/wsl/verify-appbar-installed-extensions.sh
```

It is executed by:

```bash
./scripts/wsl/preflight-v23.1.sh
```

The guard verifies:

```text
extensions/ contains only apiforge
extensions/extensions.manifest.json exposes only apiforge
APP BAR contains exactly one Extensions card
removed extension titles do not exist in frontend source/dist
```

## Runtime model

NebulaOps core starts with:

```bash
./scripts/wsl/start.sh --rebuild
```

APIForge is controlled from the UI after NebulaOps is running:

```text
http://nebulaops.localhost/api/extensions/console
```

APIForge runtime URL through gateway:

```text
http://nebulaops.localhost/apiforge/
```

No mock data or static extension records are introduced.
