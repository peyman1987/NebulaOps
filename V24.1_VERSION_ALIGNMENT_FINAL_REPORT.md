## Package metadata alignment

The Angular shell and every MFE `package.json` now expose `24.1.0`. The corresponding `package-lock.json` root metadata has also been aligned with each local package identity, avoiding copied package-lock names from another remote.

## Maven and extension alignment

Backend Maven modules are aligned to `24.1.0`. The APIForge extension Maven artifact and Dockerfile were also aligned so the built jar name matches the v24.1 release identity.

## Docker and Compose alignment

The Docker Compose project name remains `nebulaops-v24-1`, and NebulaOps-owned image names use the `nebulaops-v24-1-*` convention.

## New validation guard

A dedicated guard was added:

```bash
python3 scripts/validate-version-alignment-v24.1.py
./scripts/wsl/smoke-version-alignment-v24.1.sh
```

The static preflight now runs the version-alignment guard so future stale release identifiers fail before runtime startup.

## No mock/static data impact

This upgrade changes release identity, metadata and validation only. It does not introduce mock data or static runtime datasets.
