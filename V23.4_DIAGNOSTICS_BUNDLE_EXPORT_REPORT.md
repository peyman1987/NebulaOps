# NebulaOps v23.4 — Diagnostics Bundle Export

## Scope

This upgrade adds a support/debug export for the live NebulaOps runtime while keeping the public release version aligned to `23.4 / 23.4.0`.

## New endpoints

- `GET /api/runtime/diagnostics/bundle`
- `GET /api/runtime/diagnostics/bundle.zip`

The JSON endpoint returns the bundle manifest and embedded section payloads. The ZIP endpoint returns a downloadable archive with the same manifest plus individual files grouped by runtime domain.

## Bundle contents

The export is assembled at request time from live runtime sources only:

- Docker containers
- Docker images
- Docker networks
- Docker events
- Docker compose project status
- Docker container log tails
- Kubernetes pods
- Kubernetes events
- `kubectl describe` output for pods, deployments, services and ingress
- Helm releases
- Gateway health generated during the request
- Extension status derived from Docker runtime containers
- Frontend remote verification output
- v23.4 preflight output

## Live-only behavior

No dataset is embedded in the source code or frontend. If Docker, kubectl, Helm or local scripts are unavailable, the related file remains in the bundle with explicit command status, exit code, stdout/stderr and runtime error details.

## UI integration

The Operational Issues remote now exposes a `Download diagnostics bundle` action. The button downloads the ZIP through an authenticated `fetch` request, so it works even when the gateway API requires a Bearer token.

## Validation

A new smoke script validates both the JSON manifest and the ZIP file structure:

```bash
./scripts/wsl/smoke-diagnostics-bundle-v23.4.sh
```
