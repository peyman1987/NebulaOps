# NebulaOps v23.2 Runtime Stability Upgrade Report

## Scope
NebulaOps v23.2 focuses on runtime stability, compact operations UX, extension control reliability and release hygiene.

## Implemented changes
- Global version alignment to `23.2.0`, `v23.2`, `v23-2` and `v23_2` where applicable.
- Extension images, registry container names, frontend runtime cache keys and Keycloak/session storage keys are aligned to v23.2.
- `/api/extensions` remains a fast, non-blocking installed-extension registry endpoint.
- `/api/extensions/{slug}/status` uses short runtime probes and explicit unavailable states instead of long blocking calls.
- Added `/api/runtime/diagnostics` for Docker, Kubernetes, gateway and extension-control diagnostics with real tool output only.
- Fixed the command runner timeout path so unavailable tools cannot hold the gateway request thread indefinitely.
- Added compact shell density rules for better visibility at browser zoom levels around 75%.
- Replaced remaining visible Italian strings in the shell and Spring MVC add-on with English text.
- Updated Keycloak theme branding from legacy release to N23.2.

## Runtime contract
No mock/static operational records were introduced. When Docker, Kubernetes, Keycloak or another runtime dependency is unreachable, the API returns explicit `UNAVAILABLE`, `NOT_CONFIGURED`, `DOWN` or diagnostic tool status values.
