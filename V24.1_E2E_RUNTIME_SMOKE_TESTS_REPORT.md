# NebulaOps v24.1 — E2E Runtime Smoke Tests Report

## Scope

This upgrade adds runtime smoke tests for v24.1 without changing the public version number. The scripts validate the actual running NebulaOps gateway, frontend remotes and extension control plane instead of relying on bundled snapshots or static fixture data.

## Added scripts

```bash
./scripts/wsl/smoke-runtime-v24.1.sh
./scripts/wsl/smoke-docker-openlens-v24.1.sh
./scripts/wsl/smoke-extensions-v24.1.sh
```

Shared helpers and browser tooling:

```bash
./scripts/wsl/lib/smoke-runtime-lib.sh
./scripts/wsl/tools/e2e-ui-console-check-v24.1.mjs
```

## Runtime contracts checked

The runtime smoke verifies that core endpoints return JSON contracts and do not regress to HTML error pages or gateway timeout responses. It also runs the live-only guard so bundled operational records, demo endpoints and generated frontend rows are rejected before runtime checks continue.

Covered gateway areas:

- gateway health
- runtime diagnostics
- diagnostics bundle manifest
- Operational Issues endpoints
- extension registry
- Docker runtime status and collections
- Kubernetes namespaces, pods, events and Helm releases
- Safe Action Plan endpoints

Unavailable integrations are accepted only when the response explicitly reports a runtime state such as `UNAVAILABLE`, `DEGRADED`, `NOT_CONFIGURED`, `TIMEOUT` or a populated `toolStatus` section.

## Docker Desktop and OpenLens checks

The Docker/OpenLens smoke validates two layers:

1. same-origin `remoteEntry.js` bundles are served as JavaScript and not fallback HTML;
2. `/remotes/docker-desktop/` and `/remotes/openlens-kubernetes/` are opened in a real headless Chromium/Chrome session through the Chrome DevTools Protocol.

The browser check fails on:

- `console.error`
- runtime exceptions
- failed document/script/stylesheet requests
- JavaScript error text rendered into the UI
- empty page body after load

Backend endpoints used by both views are also checked as JSON contracts.

## Extension async control-plane checks

The extension smoke verifies that `/api/extensions` does not block or return HTTP 504. It then performs a real asynchronous start request by default:

```text
POST /api/extensions/{slug}/start
```

The expected contract is:

```text
202 Accepted
operationId
operationUrl
eventsUrl
phase
state
accepted: true
```

The script then polls the operation endpoint and confirms that the extension event stream contains the generated operation id.

Set this variable only when a non-mutating check is required:

```bash
NEBULAOPS_SKIP_EXTENSION_START_SMOKE=1 ./scripts/wsl/smoke-extensions-v24.1.sh
```

## Browser requirement

The UI console smoke uses an installed Chromium/Chrome-compatible browser. If the executable is not auto-detected, set:

```bash
CHROME_BIN=/path/to/chromium ./scripts/wsl/smoke-docker-openlens-v24.1.sh
```

For CI jobs that only validate backend contracts, browser validation can be skipped explicitly:

```bash
NEBULAOPS_ALLOW_BROWSER_SKIP=1 ./scripts/wsl/smoke-docker-openlens-v24.1.sh
```

## Live-only policy

No mock or static operational dataset was added. The scripts validate live endpoint contracts and treat unavailable runtime integrations as valid only when they return explicit state and evidence.
