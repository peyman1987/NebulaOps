# NebulaOps v23.4 Auth Bridge Runtime Fix

This package hardens the frontend/MFE runtime repair path against stale Docker images and stale same-origin `remoteEntry.js` bundles.

## Fixes

- `repair-v23.4-frontend-remotes.sh` now removes old NebulaOps frontend/MFE containers left by previous compose project names.
- Frontend/MFE repair defaults to `--no-cache --pull` because stale `remoteEntry.js` is a correctness issue.
- The repair script compares the served `remoteEntry.js` with the packaged local `dist/browser/remoteEntry.js` and warns when the frontend image is stale.
- Added `scripts/wsl/reset-v23.4-frontend-runtime.sh` for a hard reset of only the frontend and MFE runtime layer.
- Added Docker build cache-buster args for the frontend shell and all MFE Nginx images.

## Recommended recovery command

```bash
./scripts/wsl/reset-v23.4-frontend-runtime.sh
./scripts/wsl/start.sh --skip-preflight
```

For a full rebuild:

```bash
./scripts/wsl/start.sh --rebuild
```
