# NebulaOps v23.1 - APP BAR Card Footer Fix

## Issue
The APP BAR launcher card metadata row, especially the `EXTENSIONS` tag and port label, could render visually outside the card border when the description text was long.

## Fix
The launcher card layout has been hardened so that the footer always remains inside the card:

- `.launcher-card` now uses `position: relative`, `box-sizing: border-box` and `overflow: hidden`.
- The footer row is anchored inside the card with `position: absolute; bottom: 12px`.
- The card receives reserved bottom padding for the footer.
- Long subtitles are clamped to three lines to avoid pushing metadata outside the card.
- Tag and port labels use `white-space: nowrap`.

## Files changed

- `frontend/src/app/app.component.css`
- `frontend/dist/nebulaops/browser/main-3HWSZXCO.js`

## Scope
This is a UI-only fix. Kubernetes manifests, Spring Boot extensions, live endpoints and deployment scripts were not changed.
