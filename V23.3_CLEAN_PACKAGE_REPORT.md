# NebulaOps v23.3 Clean Package Report

This package is a clean v23.3 workspace intended to be extracted into a new directory, avoiding stale files left behind by older releases.

## Cleanup completed

- Removed obsolete v22 compatibility assets from `frontend/src/assets`.
- Removed obsolete v22 diagram artifacts from `docs/diagrams`.
- Removed obsolete `docker-compose.v22.5.override.yml`.
- Removed stale v22 generated PDF files from `docs/final`.
- Removed packaged local kubeconfig material from `.kube/config`; startup regenerates it from the operator environment.
- Sanitized all frontend `package-lock.json` files so resolved package URLs use the public npm registry instead of build-environment internal registry URLs.
- Added `scripts/wsl/purge-stale-release-assets.sh` and wired it into the v23.3 preflight so accidental extraction over an older folder automatically removes known stale release artifacts before validation.

## Live-only policy

Runtime consoles must use backend/runtime integrations only. Bundled mock rows, demo API endpoints, seeded operational records, personal placeholder accounts and offline snapshot assets are rejected by `scripts/verify-live-only-runtime.py`.

## Validation executed

```bash
python3 scripts/validate-package.py
python3 scripts/verify-live-only-runtime.py
./scripts/wsl/preflight-v23.3.sh
./scripts/wsl/verify-extensions-compile-inputs.sh
./scripts/wsl/verify-extensions-live-only.sh
bash -n scripts/wsl/*.sh backend/spring-mvc-service/deploy.sh
python3 -m py_compile ai-engine/app/main.py
cd frontend && npm ci --no-audit --no-fund && npm run build
```

The frontend production build completed successfully. Backend Maven reactor compilation requires a local Maven installation or Maven wrapper; the validation environment used for packaging does not provide `mvn` and the project does not ship `mvnw`.
