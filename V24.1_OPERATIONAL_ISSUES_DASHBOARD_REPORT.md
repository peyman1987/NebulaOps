# NebulaOps v24.1 Operational Issues Dashboard

## Scope

NebulaOps v24.1 adds the first Priority 1 upgrade: a live Operational Issues Dashboard that aggregates runtime issues across Docker Engine, Kubernetes, Helm and the extension control plane.

## Backend API

New Spring Boot gateway controller:

- `GET /api/platform/issues`
- `GET /api/platform/issues/summary`
- `GET /api/platform/issues/{id}/evidence`
- `GET /api/platform/issues/{id}/actions`

The controller derives issues from runtime calls only. It reports explicit source health when Docker, Kubernetes, Helm or extension control are unavailable.

## Issue categories

The dashboard reports the following operational categories:

- Docker source unavailable
- Docker containers unhealthy, exited, dead, restarting, created or paused
- Docker project risk rows
- Docker mount risk rows
- Docker container security audit rows
- Docker restart policy and healthcheck coverage rows
- Docker network exposure rows
- Kubernetes source unavailable
- Kubernetes pod phase and container-state problems
- Kubernetes warning events
- Kubernetes unhealthy node conditions
- Kubernetes PVCs not bound
- Kubernetes services without endpoint subsets
- Kubernetes ingresses without TLS sections
- Helm releases not deployed
- Extension runtime states that are not healthy

## Frontend integration

A new micro frontend was added:

- Remote id: `operational-issues`
- Custom element: `nebulaops-mfe-operational-issues`
- Route: `/remotes/operational-issues/`
- Sidebar group: `SRE`

The shell includes the new remote in the sidebar, app launcher and prebuilt Nginx dist output. Priority 2 adds row-level troubleshooting buttons rendered from backend-provided `troubleshootingActions`, including Docker logs/inspect/health, Kubernetes root-cause/connectivity, Helm status/history and extension diagnostics.

## Runtime behavior

When no issues are returned by the live sources, the API returns an empty list and the message:

`No live operational issues returned by the runtime sources.`
