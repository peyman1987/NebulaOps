# NebulaOps v23.3 — Docker/OpenLens Operational Console Step 4

## Scope

This internal v23.3 upgrade extends Docker Desktop and OpenLens Kubernetes operational consoles without changing the release version. All views are backed by live Docker Engine API calls, `kubectl`, or `helm`/cluster commands. No mock, sample, seeded, or static operational datasets are introduced.

## Docker Desktop additions

New read-only operational endpoints:

- `GET /api/runtime/docker/container-security-audit`
- `GET /api/runtime/docker/image-hygiene-audit`
- `GET /api/runtime/docker/network-inventory`
- `GET /api/runtime/docker/unused-resource-candidates`

### Behavior

- Container security audit is derived from Docker inspect data and reports privileged mode, host namespaces, writable root filesystem, root user, added capabilities, and security options.
- Image hygiene audit is derived from live image/container references and reports dangling images, `latest`-only tags, and images not referenced by current containers.
- Network inventory is derived from Docker networks and container attachments.
- Unused resource candidates is preview-only and never removes resources.

## OpenLens Kubernetes additions

New read-only operational endpoints:

- `GET /api/kubernetes/metrics/nodes`
- `GET /api/kubernetes/metrics/pods?namespace=all`
- `GET /api/kubernetes/autoscaling/hpa?namespace=all`
- `GET /api/kubernetes/availability/pdb?namespace=all`
- `GET /api/kubernetes/workload-owners?namespace=all`
- `GET /api/kubernetes/api-resources`

### Behavior

- Node and pod metrics are returned only when `kubectl top` succeeds. If metrics-server is not installed, the endpoint returns an explicit unavailable state and an empty `items` list.
- HPA and PDB summaries are collected from live Kubernetes API resources through kubectl.
- Workload owners are derived from pod `ownerReferences`.
- API resource discovery is derived from `kubectl api-resources`.
