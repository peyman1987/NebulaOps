# NebulaOps v23.2 MFE Container Sync Runtime Fix

This package fixes the case where local MFE dist artifacts and the frontend container filesystem are valid, but `nebulaops.localhost/remotes/<slug>/remoteEntry.js` is still served from stale MFE runtime containers or an older Nginx route.

Changes:

- The frontend repair now copies the current `frontend/nginx.conf` into the running frontend container before reload.
- The repair also syncs every running `mfe-*` container from its local `frontend/remotes/<slug>/dist/browser` artifact.
- A runtime owner marker verifies that `nebulaops.localhost` is really served by the current frontend container.
- Verification checks both the frontend container copy and individual MFE container copy against local SHA-256 values.
- Curl probes now use no-cache headers and cache-buster query parameters.

No mock/static operational data was added. The fix only changes runtime synchronization and verification scripts.
