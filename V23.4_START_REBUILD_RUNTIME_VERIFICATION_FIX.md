# NebulaOps v23.4 start.sh --rebuild runtime verification fix

This package fixes the repeated `./scripts/wsl/start.sh --rebuild` failure where the stack compiled and containers started, but the startup script failed because several served MFE `remoteEntry.js` files did not contain the inline auth bridge marker.

The v23.4 runtime supports two valid auth-bridge delivery modes:

1. inline bridge prepended to `remoteEntry.js`;
2. separate `nebulaops-auth-bridge.js` loaded by the shell and by standalone remote pages.

The previous startup verification accepted only mode 1. The corrected startup flow now validates that each MFE is shell-compatible and that an auth bridge is reachable either inline or as the colocated/shell asset. It still rejects missing bundles, ESM bundles, and non-custom-element remote bundles.

No mock/static operational data was added.
