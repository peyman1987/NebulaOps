# NebulaOps v23.4 — Kubernetes Network Monitoring Graph Improvement Report

## Scope

This upgrade improves the Kubernetes Network Monitoring graph without increasing the public product version. The package remains NebulaOps 23.4 / 23.4.0.

## Backend changes

- Enhanced `GET /api/kubernetes/network/graph?namespace=all`.
- The endpoint now returns graph-level findings derived only from live Kubernetes resources.
- Added topology health: `OK`, `WARN`, or `ERROR`.
- Added namespace list, node count, edge count, error count and warning count.
- Added service readiness enrichment: selected pods, ready endpoints, not-ready endpoints, service ports, selectors, external IPs and load balancer status.
- Added endpoint-to-pod target edges when Kubernetes Endpoint target references are available.
- Added missing ingress backend detection using live Ingress references.
- Added NetworkPolicy selection analysis and warning when a policy currently selects no pods.
- Added pod phase severity so problematic pods are visible in the graph.

## Frontend changes

- Improved the OpenLens Kubernetes network graph layout with curved edges, stronger severity highlighting and larger graph canvas.
- Added graph filters for namespace, node type, status and free-text search.
- Added graph health badge, filtered node/edge counters and click-to-inspect node details.
- Added graph findings panel beside the visual topology.
- Kept the existing connectivity summary panel for service-level checks.
- The frontend still does not create fallback/mock nodes. Missing service nodes are rendered only when the backend reports a real reference from a live Ingress object.

## Live-only data policy

No mock/static operational records were introduced. The graph is built from live `kubectl` responses returned by the gateway. When Kubernetes is unavailable, the UI surfaces `UNAVAILABLE`, tool status and explicit backend errors instead of fabricating topology data.

## Versioning

Public version remains `23.4.0`.
