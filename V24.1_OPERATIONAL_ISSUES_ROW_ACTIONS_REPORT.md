# NebulaOps v24.1 Operational Issues Row-Level Troubleshooting Actions

## Scope

This upgrade implements Priority 2 for v24.1 while keeping the public release version at `24.1 / 24.1.0`.

The goal is to make the Operational Issues dashboard actionable at row level. Each issue returned by `/api/platform/issues` now carries a live troubleshooting action catalog. The frontend renders those actions directly on each issue row and executes only NebulaOps runtime endpoints.

## Backend changes

`PlatformIssuesController` now exposes:

- `GET /api/platform/issues`
- `GET /api/platform/issues/summary`
- `GET /api/platform/issues/{id}/evidence`
- `GET /api/platform/issues/{id}/actions`

Every issue row now includes:

- `evidenceEndpoint`
- `actionsEndpoint`
- `troubleshootingActions`
- `actionCount`
- `realDataOnly: true`

The action catalog is derived from the real issue type and the live evidence object. No static issue rows or frontend-generated runtime records are introduced.

## Row-level action coverage

Docker container issues expose:

- Details
- Logs
- Log analysis
- Health
- Inspect

Docker project issues expose:

- Failure analysis
- Project logs
- Dependency map
- Startup order
- Health report

Kubernetes pod issues expose:

- Root cause
- Diagnostics
- Logs
- Describe

Kubernetes workload issues expose:

- Root cause
- Detail
- Dependency map
- Rollout status where applicable

Kubernetes service issues expose:

- Connectivity
- YAML
- Describe

Kubernetes ingress issues expose:

- Connectivity
- YAML
- Describe

Kubernetes namespace and PVC issues expose namespace-level diagnostic actions.

Helm release issues expose:

- Status
- Values
- History

Extension issues expose:

- Diagnostics
- Status

## Frontend changes

The generated live-only remote template now has a dedicated `operational-issues` console.

The dashboard renders:

- issue severity
- issue title and resource type
- runtime source
- message and recommended action
- per-row troubleshooting buttons from `troubleshootingActions`
- live source health cards
- raw API response in a collapsed details block

The remote still reads from `/api/platform/issues`; no issue table is assembled from bundled static data.

## Version handling

The release remains v24.1. Internal cache keys were updated to avoid stale remoteEntry loading:

- shell remote entry build key: `v24.1.2-operational-issue-actions`
- generated remote bundle key: `v24.1.2-row-actions-live-real-data-control-plane`

These are cache-busting identifiers only and do not change the public product version.
