# NebulaOps v23.4 — Docker/OpenLens Operational Console Step 3

## Scope

This internal v23.4 upgrade extends Docker Desktop and OpenLens with day-2 operational diagnostics while keeping the same release number and the live-only runtime policy.

## Docker Desktop additions

- `/api/runtime/docker/resource-pressure`
- `/api/runtime/docker/project-risks`
- `/api/runtime/docker/build-cache`
- `/api/runtime/docker/volume-usage`
- `/api/runtime/docker/containers/{id}/details`
- `/api/runtime/docker/containers/{id}/health`
- `/api/runtime/docker/containers/{id}/changes`

All Docker data is derived from Docker Engine API responses through the Unix socket. No bundled operational records are generated in the frontend or backend.

## OpenLens additions

- `/api/kubernetes/storage/summary?namespace=all`
- `/api/kubernetes/network/summary?namespace=all`
- `/api/kubernetes/namespaces/{ns}/quotas`
- `/api/kubernetes/workloads/{kind}/{ns}/{name}/rollout-status`
- `/api/kubernetes/pods/{ns}/{name}/diagnostics`

All OpenLens data is derived from `kubectl` responses against the selected cluster context. If the cluster/tooling is unavailable, endpoints return explicit unavailable/error state instead of fallback rows.

## Live-only verification

The live-only guard was executed after the upgrade and reported no bundled mock/static operational records, demo endpoints, personal placeholders or unsafe command-runner patterns.
