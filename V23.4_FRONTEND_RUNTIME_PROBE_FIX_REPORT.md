# NebulaOps v23.4 Frontend Runtime Probe Fix

This package fixes the v23.4 frontend repair flow that could report `frontend shell not ready after 90s` after rebuilding the shell and MFE images.

## Changes

- Frontend/MFE repair no longer deletes every container whose name contains `nebulaops`.
- Only stale `frontend` and `mfe-*` containers from previous compose projects are removed.
- Backend, Keycloak, Grafana, Prometheus, MongoDB, Redis and RabbitMQ are preserved during frontend repair.
- The shell readiness timeout has been increased to 180 seconds.
- Shell readiness now probes both `NEBULAOPS_PUBLIC_URL` and localhost on the configured HTTP port.
- On failure, the repair script prints port ownership, frontend compose status, frontend logs, Nginx validation, internal container probe and host probe results.
- RemoteEntry verification uses the first reachable shell base URL, preventing false negatives caused by WSL hostname resolution issues.

## Operational goal

The repair path remains strict: if the browser-facing shell is not reachable, startup stops. The difference is that it now reports the concrete container or port reason instead of only saying the shell is not ready.
