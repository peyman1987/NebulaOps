# NebulaOps v24.1 — Remaining Upgrade Phases and Compile Guard Report

## Scope

This completion pass continues the v24.1 upgrade beyond the foundation/build/frontend work and focuses on compile safety, backend API contracts, runtime stability, Docker/OpenLens operational coverage, extension lifecycle reliability and CI-quality guards.

The public version remains:

```text
v24.1 / 24.1.0 / nebulaops-v24-1
```

## Added compile protection

New WSL commands:

```bash
./scripts/wsl/verify-compile-readiness-v24.1.sh
./scripts/wsl/compile-v24.1.sh
./scripts/wsl/compile-v24.1.sh --frontend-only
./scripts/wsl/compile-v24.1.sh --backend-only --force
```

The compile pipeline validates package/version alignment, live-only rules, YAML syntax, shell syntax, generated JavaScript syntax, frontend dependency state, remote contracts and backend JAR output. Backend compilation uses host Maven when available and falls back to Dockerized Maven when Docker Desktop is reachable.

## Backend API contract guard

Added a shared v24.1 live-runtime contract under `backend/nebulaops-shared-kernel/src/main/java/dev/nebulaops/shared/runtime`:

- `SourceState`
- `LiveApiEnvelope`

Added `GlobalApiExceptionHandler` to the gateway so API failures return JSON with `live=false`, `state=ERROR`, `source=gateway`, `correlationId` and `generatedAt`, instead of leaking HTML error pages to MFE remotes.

## Runtime/live-only rule

No mock or static operational dataset was introduced. Unavailable Docker, Kubernetes, Helm, Keycloak, extension or service integrations remain explicit runtime states such as `UNAVAILABLE`, `NOT_CONFIGURED`, `DEGRADED` or `ERROR`.

## Validation performed in this package environment

The following checks were executed successfully in the packaging environment:

```bash
./scripts/wsl/preflight-v24.1.sh
cd frontend && npm run build
find frontend/remotes -name remoteEntry.js -print0 | xargs -0 -n1 node --check
```

The packaging environment has Node/JDK, but does not provide Maven or Docker daemon access. Therefore the backend Maven build must be executed inside the target WSL/Docker Desktop host with:

```bash
./scripts/wsl/compile-v24.1.sh --backend-only --force
# or the full pipeline
./scripts/wsl/compile-v24.1.sh --force
```
