# NebulaOps v23.3 — Docker/OpenLens Operational Console Step 2

## Scope

This upgrade stays inside the v23.3 release line. It extends the existing Docker Desktop and OpenLens Kubernetes operational consoles without introducing mock records, sample data, static operational fallbacks or fabricated health states.

## Docker Desktop improvements

### Compose project control

New live Docker Engine endpoints:

- `GET /api/runtime/docker/projects/{project}`
- `POST /api/runtime/docker/projects/{project}/start`
- `POST /api/runtime/docker/projects/{project}/stop?t=10`
- `POST /api/runtime/docker/projects/{project}/restart?t=5`
- `GET /api/runtime/docker/projects/{project}/logs?tail=120`

The project view is derived from real Docker labels such as `com.docker.compose.project`, `com.docker.compose.service` and `com.docker.stack.namespace`.

### Port conflict detection

New live endpoint:

- `GET /api/runtime/docker/port-conflicts`

The detector reads published ports from Docker container metadata and reports only actual conflicts where more than one container reports the same host port.

### UI changes

The Docker Desktop MFE now exposes:

- Compose project actions: logs, start, restart, stop
- published port conflict panel
- richer project drill-down through real backend endpoints

## OpenLens Kubernetes improvements

### Port-forward manager

New endpoints:

- `GET /api/kubernetes/port-forwards`
- `POST /api/kubernetes/port-forwards`
- `DELETE /api/kubernetes/port-forwards/{id}`

The manager creates real `kubectl port-forward` processes bound to `127.0.0.1`, stores metadata under the system temp folder and exposes running/stopped state from the actual process status.

### Helm lifecycle operations

New endpoints:

- `GET /api/kubernetes/helm/releases/{namespace}/{name}/status`
- `GET /api/kubernetes/helm/releases/{namespace}/{name}/values`
- `GET /api/kubernetes/helm/releases/{namespace}/{name}/history`
- `POST /api/kubernetes/helm/releases/{namespace}/{name}/rollback`
- `DELETE /api/kubernetes/helm/releases/{namespace}/{name}`

These endpoints execute real Helm commands through the cluster-aware command path. Rollback and uninstall are mutation operations and return stdout, stderr, exit code, duration and mutation correlation metadata.

### RBAC explorer

New endpoints:

- `GET /api/kubernetes/rbac/summary?namespace=all`
- `GET /api/kubernetes/rbac/can-i?verb=list&resource=pods&namespace=default`

The summary aggregates live `kubectl get` results for ServiceAccounts, Roles, RoleBindings, ClusterRoles and ClusterRoleBindings. The `can-i` endpoint delegates to `kubectl auth can-i` and returns the actual authorization answer.

### UI changes

The OpenLens MFE now exposes:

- active port-forward list and stop action
- port-forward creation form
- Helm action panel: status, values, history, rollback and uninstall
- RBAC summary panel
- `kubectl auth can-i` explorer

## Hardening fixes

- Avoided blank-path deletion when stopping a port-forward.
- Copied named kubeconfig material into the managed port-forward folder and removed the temporary source file after copy.
- RBAC summary now reports unavailable resources explicitly instead of claiming live status when `kubectl` calls fail.
- Shell quoting for cluster-specific kubeconfig paths and impersonation users is handled by a dedicated helper.

## Environment limitation

This sandbox does not provide Maven, Docker daemon or a Kubernetes cluster. The Maven backend build and runtime checks must be executed in the target WSL/Docker Desktop environment.

Recommended target-environment commands:

```bash
./scripts/wsl/build-backend-jars.sh --force
./scripts/wsl/start.sh --rebuild
curl http://nebulaops.localhost/api/runtime/docker/port-conflicts
curl http://nebulaops.localhost/api/kubernetes/port-forwards
curl 'http://nebulaops.localhost/api/kubernetes/rbac/can-i?verb=list&resource=pods&namespace=default'
```
