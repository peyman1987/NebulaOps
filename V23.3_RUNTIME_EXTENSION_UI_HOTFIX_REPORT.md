# NebulaOps v23.3 Runtime Extension and UI Hotfix

## Fixes

- `/api/extensions` is now a fast registry endpoint and does not run blocking kubectl probes for every installed extension.
- Deep live status remains available through `/api/extensions/{slug}/status`.
- Extension list remains real and explicit: installed extensions are returned as disabled by default, without mock runtime state.
- Updated remaining legacy runtime version references to v23.3 where they were visible or executable.
- UI strings found in Italian in the shell and extensions panel were converted to English.
- Added compact presentation density so the UI remains readable at normal browser zoom while preserving the improved visibility observed at 75% zoom.

## Operational note

If `localhost:8088` or `localhost:8089` returns connection refused, Mongo Express or Redis Commander is not running or the optional profile was not started. The platform should prefer `http://nebulaops.localhost` links and runtime readiness should report those native tools explicitly as unavailable instead of leaving the browser with a raw connection-refused page.


## Native tool port fix

- `mongo-express` now exposes `localhost:8088` consistently.
- `redis-commander` now exposes `localhost:8089` consistently.
- This matches the startup summary and prevents raw Chrome `ERR_CONNECTION_REFUSED` pages when users open the native Mongo/Redis UI links.
