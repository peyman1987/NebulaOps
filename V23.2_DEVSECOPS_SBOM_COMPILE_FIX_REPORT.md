# NebulaOps v23.2 — DevSecOps SBOM Compile Fix

## Issue

The Maven build failed in `devsecops-service` because `DevSecOpsController` called:

```java
service.sbom(image)
```

but `DevSecOpsService` did not expose a matching `sbom(String)` method.

## Fix

Implemented `DevSecOpsService.sbom(String image)` using real Trivy CycloneDX output:

```bash
trivy image --format cyclonedx --quiet <image>
```

No mock/static SBOM data is returned. If the image is missing or Trivy cannot execute, the endpoint returns an explicit non-live status with `toolStatus`.

## API improvement

The existing endpoint remains available:

```text
GET /api/devsecops/sbom/{image}
```

A safer endpoint was added for real container image references that contain `/`, such as `registry.local/team/app:tag`:

```text
GET /api/devsecops/sbom?image=<registry/image:tag>
```
