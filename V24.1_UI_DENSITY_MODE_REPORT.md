# NebulaOps v24.1 — UI/UX Density Mode Report

## Scope

This upgrade keeps the public version at **24.1 / 24.1.0** and focuses on visual density, hierarchy and operator readability without changing the application style or introducing static runtime data.

## Implemented changes

### Shell density

- Added `frontend/src/assets/nebulaops-density-v24-1.css`.
- Added the same stylesheet to `frontend/dist/nebulaops/browser/assets/nebulaops-density-v24-1.css` so the already-built shell uses the compact layout immediately.
- Linked the stylesheet from both source and prebuilt shell `index.html` files.
- Reduced main shell spacing, header height, sidebar item height and remote frame chrome.
- Kept the active MFE identity and endpoint path visible while reducing vertical overhead.
- Made the shell header sticky so the operator keeps context while scrolling dense runtime views.

### Remote MFE density

- Added compact density CSS to `frontend/tools/live-only-remote.template.js` for future generated remotes.
- Patched all generated `frontend/remotes/*/remoteEntry.js` files, including `dist/browser/remoteEntry.js` copies.
- Reduced hero height, card padding, table cell padding, button size and panel gaps.
- Added sticky panel headers and sticky table headers for large runtime tables.
- Kept endpoint status next to the active title/filter area.
- Kept filters visible through sticky panel and network toolbar behavior.

### Detail drawer

- Replaced the previous full-width detail panel with a right-side drawer.
- The drawer is fixed, scrollable and independent from the table scroll area.
- Row-level troubleshooting, safe action plans and diagnostics results now remain visible beside the list instead of pushing the table down.

### Raw responses

- Raw endpoint responses are now closed by default.
- Details blocks remain available for technical inspection but do not consume screen height on first render.

### Extension Control Plane

- Added compact modal density for the extension control panel.
- Reduced header, toolbar, list, metrics, operation phases and log spacing.
- Preserved the async operation lifecycle from priority 4.

## Live-only guarantee

The upgrade is visual/layout only. It does not add operational records, frontend-generated rows, demo endpoints, fallback incidents, container lists, pod lists or Kubernetes snapshots.

Runtime views continue to rely on live NebulaOps gateway endpoints and explicit unavailable/degraded states.
