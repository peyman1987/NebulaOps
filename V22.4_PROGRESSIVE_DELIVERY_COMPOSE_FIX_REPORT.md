# NebulaOps v22.4 Progressive Delivery Compose Fix Report

## Issue fixed

`./scripts/wsl/start.sh --rebuild` failed during Docker Compose project resolution with:

```text
service "ai-ops-service" depends on undefined service "progressive-delivery-service": invalid compose project
```

The Maven reactor build was successful and all backend JARs were produced. The failure was caused by Docker Compose wiring: `ai-ops-service` referenced `progressive-delivery-service` in `depends_on`, while the top-level Compose service definition was missing.

## Fixes applied

- Added top-level `progressive-delivery-service` to `docker-compose.yml`.
- Added top-level `progressive-delivery-service` to `infrastructure/docker-compose.yml`.
- Added runtime configuration for Keycloak, audit, notification, Argo CD and tool path access.
- Added port mapping `8102:8102`.
- Added healthcheck against `/actuator/health`.
- Kept runtime-only progressive delivery behavior: no mock/static data.
- Hardened `scripts/validate-yaml.py` so preflight now fails if any Compose service uses `depends_on` with an undefined service.

- `scripts/wsl/preflight-v22.4.sh` OK.
