# NebulaOps v23.1 Implementation Report

This package upgrades NebulaOps from v23.1 to v23.1 with live-only operational modules. The changes intentionally avoid fake records, seeded UI rows and static runtime states.

## Files changed with functional impact
- `frontend/remotes/*/live-endpoints.json`: redesigned remote source mapping for Docker Desktop, OpenLens Kubernetes, INFRA Hub, Task Management, Observability, Release Center, Progressive Delivery, Policy Center and Identity Admin.
- `frontend/tools/generate-live-only-remote.cjs`: versioned remote generator for v23.1.
- `frontend/tools/verify-remotes.mjs`: v23.1 remote verification and JWT key validation.
- `backend/gateway-service/.../DockerSocketClient.java`: Docker socket status diagnostics.
- `backend/gateway-service/.../DockerRuntimeService.java`: live-only Docker collection envelope with explicit tool status.
- `backend/gateway-service/.../RuntimeOpsController.java`: Docker status, stats, builds and events endpoints.
- `backend/gateway-service/.../KubernetesOpsController.java`: stable `/api/kubernetes/cluster` endpoint.
- `backend/gateway-service/.../InfraRuntimeController.java`: new INFRA Hub runtime endpoints.
- `backend/observability-service/.../ObservabilityService.java`: incident timeline correlation from live sources.
- `backend/observability-service/.../ObservabilityServiceController.java`: incident timeline endpoint.
- `backend/auth-service/.../IdentityAdminController.java`: realm status and realm cache status endpoints.
- `backend/auth-service/.../IdentityCacheController.java`: global Redis identity cache status endpoint.
- `backend/auth-service/.../KeycloakIdentityAdminService.java`: Keycloak realm availability classification.
- `backend/auth-service/.../RedisIdentityCache.java`: Redis cache health check.
