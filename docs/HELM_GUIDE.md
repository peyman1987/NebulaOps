# Helm Guide

## Purpose

The Helm chart packages NebulaOps for Kubernetes-style deployment. It provides a clear separation between templates,
values and environment-specific configuration.

## Render manifests locally

```bash
./scripts/helm-render.sh
```

Or directly:

```bash
helm template nebulaops ./helm/nebulaops
```

## Install locally into a Kubernetes cluster

```bash
./scripts/helm-install-local.sh
```

## Recommended chart structure

```text
helm/nebulaops/
  Chart.yaml
  values.yaml
  templates/
```

## Production recommendations

- define resource requests and limits
- use Kubernetes secrets for credentials
- configure probes for every service
- separate values per environment
- add ingress/TLS configuration
- externalize MongoDB, Redis and RabbitMQ for production
- use image tags generated by CI
