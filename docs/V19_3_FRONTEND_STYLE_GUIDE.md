# NebulaOps v19.3 — Frontend Style Guide

La v19.3 usa un cockpit Angular dark/cyber con card glassmorphism, griglie olografiche, glow controllato e animazioni
operative.

## Tab principali

- `AI OPS` — incident cockpit, RCA visuale, dependency graph e AUTO FIX demo-safe.
- `KUBERNETES` — topology 3D live con nodes, pods, services, ingress e volumes.
- `SECURITY` — radar animation, threat map e critical alerts.
- `COMPLIANCE` — policy posture, score e controlli simulati.
- `VULNERABILITIES` — CVE dashboard, severity breakdown e dependency findings.

## Icone e colore originale

Le icone tecnologiche devono restare grandi e leggibili, usando colori coerenti con il brand:

- Kubernetes: blu.
- Terraform: viola.
- Helm: navy/turchese.
- Grafana: orange/pink.
- Security/DevSecOps: rosso, amber e gold per alert/risk.

## DevSecOps visual language

- Radar sweep animato per scansioni.
- Threat map con nodi pulsanti.
- Risk score animato con gradient cyber.
- Critical alerts con glow rosso controllato.
- CVE table/cards con severity `CRITICAL`, `HIGH`, `MEDIUM`, `LOW`.

## Kubernetes Visual Cluster

- Connessioni animate tra services/pods/nodes.
- CPU/RAM bars animate.
- Restart animation per pod instabili.
- Pod detail panel: logs live, terminal embedded, metrics, events, YAML live.

## Accessibilità

- Contrasto alto su dark background.
- Stato operativo leggibile anche senza colore.
- Animazioni continue ma leggere, senza bloccare l'interazione.
