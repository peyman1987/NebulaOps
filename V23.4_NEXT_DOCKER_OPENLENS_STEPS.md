# NebulaOps v23.4 — Next Docker/OpenLens Steps

Keep the version fixed at **v23.4**. The next steps should continue the live-only policy: no bundled operational records, no generated fallback datasets, no static runtime examples.

## Step A — Row-level troubleshooting actions

Wire the new drill-down endpoints directly into table rows:

- Pod → Root cause
- Workload → Root cause
- Service → Connectivity
- Ingress → Connectivity
- Docker container → Log analysis
- Docker Compose project → Failure analysis

## Step B — Safe Action Plan

Before destructive or disruptive actions, show an impact plan:

- Docker project stop/restart
- Docker prune
- container kill/remove
- Kubernetes rollout restart
- scale workload
- delete pod/job/YAML
- Helm rollback/uninstall

Suggested endpoints:

```text
POST /api/runtime/docker/actions/plan
POST /api/kubernetes/actions/plan
```

## Step C — Diagnostics Bundle Export

Add exportable runtime bundles for support/troubleshooting:

```text
GET /api/runtime/diagnostics/bundle
GET /api/runtime/diagnostics/bundle.zip
```

The bundle should include live Docker snapshot, Docker logs tail, Docker events, Kubernetes pods/events/describes/rollout status and NebulaOps diagnostics.

## Step D — Operational Issues dashboard

Create a single landing panel combining Docker and Kubernetes findings:

- unhealthy containers
- project failures
- log errors
- CrashLoopBackOff/ImagePullBackOff
- service without endpoints
- ingress without TLS
- risky mounts/exposed ports

## Step E — E2E runtime smoke tests

Add Playwright or shell smoke tests for:

- Docker Desktop console loads
- OpenLens console loads
- no console errors when opening Docker/OpenLens
- troubleshooting endpoints return either live data or explicit unavailable states
- no 500/504 in core Docker/OpenLens endpoints
