# NebulaOps v23.1 — Requirements Verification and Improvement Report

Date: 2026-05-27
Package: `nebulaops-v23.1`

## Runtime data policy

NebulaOps v23.1 has been reviewed with the following rule: runtime consoles must render only live responses from backend services or external runtimes. The frontend remotes do not create fallback mock records. Empty states mean that the live endpoint returned no rows or that the integration is unavailable. Error states are rendered explicitly.

## Requirement checklist

| Area | Verification result | Improvements applied |
|---|---|---|
| Docker Desktop | Completed | Added Docker Engine socket actions for containers, images, volumes, networks, logs, stats, inspect, prune, restart, delete and system disk usage. Added explicit socket states: `DOCKER_SOCKET_NOT_FOUND`, `DOCKER_SOCKET_PERMISSION_DENIED`, `DOCKER_DESKTOP_STOPPED`, `DOCKER_ENGINE_UNREACHABLE`. |
| OpenLens Kubernetes | Completed | UI now exposes pods, deployments, services, ingress, configmaps, statefulsets, nodes, namespaces, logs, describe, YAML view/edit/apply/delete, restart, scale, cordon, uncordon and drain. Added missing StatefulSet YAML/describe and ConfigMap describe endpoints. |
| INFRA Hub | Completed | Converted into platform control center using live probes for services, ports, reverse proxy, Keycloak, RabbitMQ, Redis, MongoDB, Prometheus, Grafana, OPA, GitLab, Argo CD and installed MFE extensions discovered from Docker runtime. |
| Task Management | Completed | Added enterprise board UI, filtering by text/assignee/priority, task creation, drag/drop state changes, release/audit/notification/policy links and RabbitMQ task-event visibility. |
| Observability | Completed | Added/verified service health, Prometheus, Loki, Tempo, RabbitMQ, audit, notification and incident timeline sources. Task events now read RabbitMQ management API with `ack_requeue_true`, so messages are not consumed. |
| Release Center + Progressive Delivery | Completed | Release Center focuses on release lifecycle, gates, approvals and audit. Progressive Delivery focuses on Argo Rollouts/Argo CD operations: canary/blue-green, promote, abort, restart, history and sync. The UI avoids duplicating responsibilities. |
| Policy Center | Completed | UI now links OPA policies, governance decisions, approvals, violations, release gates, DevSecOps findings and audit. Added runtime evaluation action from Policy Center. |
| Identity Admin | Completed | UI loads users, groups, roles, realm status and Redis cache status from real Keycloak/auth endpoints. Fixed realm switching so status/cache endpoints are not accidentally rewritten to user/group/role calls. Added gateway proxy endpoints for realm status and identity cache status. |

## Important runtime notes

- Docker features require `/var/run/docker.sock` mounted/readable by `gateway-service`.
- Kubernetes features require `kubectl` and the correct kube context inside the gateway runtime.
- Progressive Delivery requires Argo Rollouts/Argo CD CLI/API availability depending on the action.
- Identity Admin requires Keycloak admin configuration and Redis availability for cache status.
- Observability timeline is useful only when Prometheus, Loki, Tempo, RabbitMQ, audit-service and notification-service are reachable.
