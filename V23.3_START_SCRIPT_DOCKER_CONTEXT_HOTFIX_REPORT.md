# NebulaOps v23.3 — Start Script Docker Context Hotfix

## Issue

After a successful backend Maven build, `./scripts/wsl/start.sh` failed with:

```text
./scripts/wsl/start.sh: line 581: .../scripts/wsl/repair-v23.3-docker-context.sh: No such file or directory
```

## Root cause

`start.sh` invoked `scripts/wsl/repair-v23.3-docker-context.sh`, but the script was not included in the packaged release.

## Fix

- Added `scripts/wsl/repair-v23.3-docker-context.sh`.
- Made the `start.sh` invocation guarded and non-fragile.
- The repair script validates that backend JARs are visible from Docker's root build context.
- The repair script protects `.dockerignore` from excluding `backend/<service>/target/*.jar`.
- The repair script writes `reports/docker-context.json`.

## Verification

Run locally:

```bash
bash -n scripts/wsl/start.sh scripts/wsl/build-backend-jars.sh scripts/wsl/repair-v23.3-docker-context.sh
./scripts/wsl/start.sh --skip-preflight
```

If backend JARs were already built successfully, startup should continue beyond the previous missing-script error.
