# NebulaOps v23.1 — Progressive Delivery Center

This extension keeps the platform on version 23.1 and adds a Progressive Delivery Center for Argo Rollouts and Argo CD operations.

## Runtime-only data policy

The Progressive Delivery Center does not ship local records. The remote reads `/api/progressive-delivery/**`, and the backend obtains data from runtime tools only:

- `kubectl get rollouts.argoproj.io`
- `kubectl get analysisruns.argoproj.io`
- `kubectl get experiments.argoproj.io`
- `argocd app list -o json`
- configured Argo CD API when `NEBULAOPS_ARGOCD_URL` and `NEBULAOPS_ARGOCD_TOKEN` are available

Empty screens mean that the live source returned no records or the corresponding tool/API is not reachable.

## Actions

- promote rollout
- full promote rollout
- abort rollout
- restart rollout
- sync Argo CD application

Each write action records an audit event through `audit-service` and sends a notification when the runtime action returns a warning state.
