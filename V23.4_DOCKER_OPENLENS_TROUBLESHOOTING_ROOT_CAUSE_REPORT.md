# NebulaOps v23.4 — Docker/OpenLens Troubleshooting & Root Cause Upgrade

## Scope

This upgrade keeps the product version fixed at **v23.4 / 23.4.0** and extends the existing Docker Desktop and OpenLens Kubernetes operational consoles with troubleshooting and root-cause analysis endpoints.

No bundled operational records were added. Every new view is backed by Docker Engine API, kubectl/helm, or explicit unavailable/not-configured runtime states.

## Docker Desktop additions

New live endpoints:

```text
GET /api/runtime/docker/projects/{project}/health-report
GET /api/runtime/docker/projects/{project}/failure-analysis
GET /api/runtime/docker/projects/{project}/startup-order
GET /api/runtime/docker/projects/{project}/dependency-map
GET /api/runtime/docker/projects/{project}/log-analysis?tail=500
GET /api/runtime/docker/containers/{id}/log-analysis?tail=500
GET /api/runtime/docker/timeline?project={project}&windowSeconds=3600
```

Capabilities:

- Compose project health report from container inspect/status/healthcheck data.
- Failure analysis combining container state, exit code, restart count, healthcheck status and optional live log evidence.
- Startup order derived from Docker container creation timestamps and Compose labels.
- Dependency map covering containers, published ports, networks and mounts.
- Log intelligence detecting exceptions, errors, warnings, connection refused, address already in use, permission denied, timeouts, missing files, OOM and authentication failures.
- Docker operations timeline derived from bounded Docker Engine `/events` calls.

## OpenLens Kubernetes additions

New live endpoints:

```text
GET /api/kubernetes/pods/{namespace}/{name}/root-cause
GET /api/kubernetes/workloads/{kind}/{namespace}/{name}/root-cause
GET /api/kubernetes/services/{namespace}/{name}/connectivity
GET /api/kubernetes/ingress/{namespace}/{name}/connectivity
GET /api/kubernetes/network/connectivity-summary?namespace=all
GET /api/kubernetes/workloads/{kind}/{namespace}/{name}/dependency-map
GET /api/kubernetes/namespaces/{namespace}/dependency-map
```

Capabilities:

- Pod RCA from `kubectl get pod`, `kubectl describe pod`, current logs, previous logs and related events.
- Workload RCA from workload JSON, rollout status, describe output, owned pods and namespace events.
- Service connectivity analyzer checking selector-to-pod matching, ready endpoints, not-ready endpoints, ingress references and network policy presence.
- Ingress connectivity analyzer checking backend services, backend endpoints and TLS configuration.
- Namespace/workload dependency map across workloads, pods, services, ingress, configmaps, secrets and PVCs.

## UI exposure

The Docker Desktop remote now exposes project health, project failure analysis, startup order, dependency map, project log analysis and operations timeline.

The OpenLens Kubernetes remote now exposes connectivity summary and the default namespace dependency map. Specific pod/workload/service/ingress troubleshooting endpoints are available from the gateway for drill-down and can be wired to row-level actions in the next UI step.
