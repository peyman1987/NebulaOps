# NebulaOps v22.5 — APP BAR / EXTENSIONS Split Fix

## Objective

Separate the generic APP BAR launcher from extension lifecycle controls.

The APP BAR is now used only for platform services, MFEs and runtime routes. Extensions are managed from a dedicated EXTENSIONS button.

## UI Changes

- Sidebar footer now contains two separate controls:
  - `App Bar`
  - `Extensions`
- APP BAR no longer contains extension lifecycle cards.
- EXTENSIONS opens a dedicated in-shell control panel.
- Removed the previous nested APP BAR action-button injection script because it created invalid nested button markup and could crash the shell when opening APP BAR.

## Installed Extensions

Only these three extensions are exposed by the EXTENSIONS panel:

- APIForge
- KubeBridge
- Contract Hub

## Extension Actions

Each extension supports:

- Start
- Stop
- Restart
- Status
- Open

The UI calls the gateway extension-control endpoints:

- `GET /api/extensions`
- `GET /api/extensions/{slug}/status`
- `POST /api/extensions/{slug}/start`
- `POST /api/extensions/{slug}/stop`
- `POST /api/extensions/{slug}/restart`

## Runtime Mode

The status panel uses only live gateway/Kubernetes state. It does not inject mock records or static runtime states.
