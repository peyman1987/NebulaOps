# NebulaOps v23.1 — Extension APP BAR UI Fix

## Scope

The APP BAR must expose only the installed extensions:

- APIForge
- KubeBridge
- Contract Hub

All previously removed extensions must not appear in the APP BAR.

## UI behavior

The APP BAR now adds direct controls on each installed extension card:

- Start
- Stop
- Restart
- Status
- Open

The controls call the gateway extension control plane:

- `POST /api/extensions/{slug}/start`
- `POST /api/extensions/{slug}/stop`
- `POST /api/extensions/{slug}/restart`
- `GET /api/extensions/{slug}/status`

The Open button opens the gateway-proxied extension URL:

- `/apiforge/`
- `/kubebridge/`
- `/contract-hub/`

## Runtime guard

A browser-side APP BAR guard was added to the frontend dist assets. It removes any stale extension cards rendered by an older bundle and enriches the installed extension cards with the control buttons.

This avoids stale browser/cache behavior during local iteration.

## Cache handling

The frontend dist main bundle was renamed and `index.html` was updated with a cache-busting URL so the browser does not keep serving the previous APP BAR bundle.
