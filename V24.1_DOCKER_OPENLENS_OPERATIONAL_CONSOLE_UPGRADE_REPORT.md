# NebulaOps v24.1 — Docker/OpenLens Operational Console Upgrade

## Scope

This upgrade keeps the release identifier at `v24.1` / `24.1.0` and improves the existing Docker Desktop and OpenLens Kubernetes remotes as operational consoles. The implementation uses only live gateway endpoints and explicit unavailable states; no mock or static operational records were introduced.

## Docker Desktop Console

### Added backend endpoints

- `GET /api/runtime/docker/projects`
  - Builds a Docker Compose project view from live Docker labels such as `com.docker.compose.project` and `com.docker.compose.service`.
  - Groups containers by project, service, image and network.
- `GET /api/runtime/docker/diagnostics`
  - Reports Docker socket/daemon status, running/exited/restarting/unhealthy containers, dangling images, disk usage and storage driver information.
- `GET /api/runtime/docker/topology`
  - Builds container-to-image, container-to-network and container-to-volume relationships from live Docker Engine data.
- `GET /api/runtime/docker/prune/preview`
  - Reads Docker `/system/df` and returns reclaimable candidates without executing prune.
- `GET /api/runtime/docker/system/info`
- `GET /api/runtime/docker/system/version`
- `GET /api/runtime/docker/containers/{id}/top`
- `POST /api/runtime/docker/containers/{id}/kill`
- `GET /api/runtime/docker/images/{id}/history`

### UI improvements

- Added Docker Compose project table.
- Added Docker diagnostics panel with live operational checks.
- Added Docker topology panel.
- Added prune preview button.
- Added container `Top` and `Kill` actions.
- Added image history action.
- Destructive actions now require explicit target confirmation in the UI.

## OpenLens Kubernetes Console

### Added backend endpoints

- `GET /api/kubernetes/problems?namespace=all`
  - Detects Kubernetes problems from live `kubectl` data.
  - Covers pod phase issues, `CrashLoopBackOff`, `ImagePullBackOff`, `ErrImagePull`, high restart count, warning events, unready nodes, pending PVCs and services without endpoint subsets.
- `GET /api/kubernetes/namespaces/{ns}/overview`
  - Aggregates namespace resources from live API calls.
- `GET /api/kubernetes/workloads/{kind}/{ns}/{name}/detail`
  - Returns JSON, YAML, describe output and related events for a selected workload.
- `POST /api/kubernetes/resources/dry-run`
  - Runs server-side dry-run with `kubectl apply --dry-run=server`.
- `GET /api/kubernetes/secrets?namespace=all`
  - Lists real Secret metadata and keys while masking all values at the gateway.
- DaemonSet actions:
  - `GET /api/kubernetes/daemonsets/{ns}/{name}/yaml`
  - `GET /api/kubernetes/daemonsets/{ns}/{name}/describe`
  - `POST /api/kubernetes/daemonsets/{ns}/{name}/restart`
- Job actions:
  - `GET /api/kubernetes/jobs/{ns}/{name}/yaml`
  - `GET /api/kubernetes/jobs/{ns}/{name}/describe`
  - `DELETE /api/kubernetes/jobs/{ns}/{name}`
- CronJob actions:
  - `GET /api/kubernetes/cronjobs/{ns}/{name}/yaml`
  - `GET /api/kubernetes/cronjobs/{ns}/{name}/describe`
  - `POST /api/kubernetes/cronjobs/{ns}/{name}/suspend`
  - `POST /api/kubernetes/cronjobs/{ns}/{name}/resume`

### UI improvements

- Added Kubernetes problem detector panel.
- Added server dry-run button in the YAML editor.
- Added DaemonSet, Job, CronJob, masked Secret, PVC and StorageClass endpoint coverage.
- Added workload detail action for pods, deployments, statefulsets, daemonsets, jobs and cronjobs.
- Added Job delete, CronJob suspend/resume and DaemonSet restart actions.
- Secret values are never rendered in the remote; the gateway masks them before returning data.

## Corrective fix included

- Removed duplicate event insertion in Kubernetes log generation. Each Kubernetes event now appears once in the derived log stream.

## Runtime verification required on WSL/Docker Desktop

The sandbox used for this package does not expose Docker daemon, Kubernetes cluster or Maven. Final runtime checks must be executed on the target WSL/Docker Desktop host:

```bash
./scripts/wsl/build-backend-jars.sh --force
./scripts/wsl/start.sh --rebuild
curl http://nebulaops.localhost/api/runtime/docker/projects
curl http://nebulaops.localhost/api/runtime/docker/diagnostics
curl http://nebulaops.localhost/api/kubernetes/problems?namespace=all
```
