# NebulaOps v22.4 Implementation Report

## Scope

NebulaOps v22.4 keeps the existing package layout and extends the current shell/MFE/runtime architecture without replacing existing modules.

## Implemented components

### Task board

- `backend/task-service` now supports task ordering and board movement.
- `frontend/remotes/task-management` renders a dedicated Kanban board from live `/api/tasks` data.
- Drag-and-drop movement calls `PATCH /api/tasks/{id}/move`.
- Task creation and deletion call the existing gateway-backed task API.
- Task events include `recipientUserId`, `assigneeId` and `notificationMessage` so the notification stream can show who receives the message.

### Identity Admin

- `backend/auth-service` now includes a Keycloak admin integration for users, groups and roles.
- Keycloak list endpoints are cached in Redis for short-lived list caching and invalidated after mutations.
- `backend/gateway-service` exposes the identity API through `/api/identity/realms/{realm}`.
- `frontend/remotes/identity-admin` renders users, groups and roles per realm and supports add, edit and disable actions.

### Runtime configuration

- `docker-compose.yml` and `infrastructure/docker-compose.yml` include `mfe-identity-admin`.
- `auth-service` receives Redis and Keycloak admin environment variables.
- `frontend/Dockerfile` copies the Identity Admin remote bundle into the shell image.
- `frontend/tools/verify-remotes.mjs` validates the new remote.

## Observability & Audit Center

Added a live-only SRE console backed by `observability-service`. The module aggregates runtime service health, observability stack probes, Prometheus, Loki, Tempo, RabbitMQ management, task records, notification records and audit-service events. No mock or static operational rows are generated by the frontend bundle.
