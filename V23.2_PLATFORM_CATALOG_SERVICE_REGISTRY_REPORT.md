# NebulaOps v23.2 — Platform Catalog & Service Registry

## Scope

This package adds a new operational section named **Platform Catalog & Service Registry**. The goal is to centralize platform component metadata and live runtime state that was previously scattered across the Angular shell, compose files, remote manifests, scripts and documentation.

## New backend endpoint

The gateway now exposes the catalog under:

```text
GET /api/platform/catalog
GET /api/platform/catalog/components
GET /api/platform/catalog/remotes
GET /api/platform/catalog/backends
GET /api/platform/catalog/dependencies
GET /api/platform/catalog/openapi
GET /api/platform/catalog/runtime
```

The endpoint returns component records for:

- micro frontends;
- backend services;
- infrastructure/runtime dependencies;
- ports and base URLs;
- health endpoint result;
- owner/service group;
- main API endpoints;
- dependencies;
- Docker service mapping;
- Kubernetes workload mapping;
- OpenAPI link;
- links to logs, metrics, traces, release and policy.

The catalog performs live probes. When a service, Docker socket or Kubernetes context is not available, the response contains explicit states such as `UNREACHABLE`, `DOCKER_SOCKET_UNAVAILABLE`, `DOCKER_CONTAINER_NOT_FOUND` or `KUBERNETES_CONTEXT_UNAVAILABLE_OR_NO_DEPLOYMENTS`. It does not return seeded/mock operational rows.

## New micro frontend

A new same-origin MFE has been added:

```text
/remotes/platform-catalog/
/remotes/platform-catalog/remoteEntry.js
```

Custom element:

```text
nebulaops-mfe-platform-catalog
```

The MFE reads only `/api/platform/catalog/**` endpoints. It does not duplicate service rows in the frontend.

## Performance improvements

- Remote endpoint refresh now runs concurrent endpoint calls instead of sequential calls in the shared live-only remote runtime.
- Gateway `RestTemplate` now uses configurable connection/read timeouts:

```text
NEBULAOPS_HTTP_CONNECT_TIMEOUT_MS default 2500
NEBULAOPS_HTTP_READ_TIMEOUT_MS default 15000
```

This prevents slow/unreachable probes from blocking the catalog and other gateway aggregation screens for too long.

## Runtime packaging updates

- Added `mfe-platform-catalog` to Docker Compose.
- Added the remote static bundle to the frontend Nginx image.
- Added the remote to shell navigation, app launcher, command palette and health checks.
- Added script checks for the new MFE in WSL startup/build/health scripts.
